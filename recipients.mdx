## Overview

Use the **Direct access link** when a specific family needs access to a form but does not match your current recipient filters. This gives that family a direct path to the form without changing the main recipient setup for everyone else.

<Callout kind="info">

The direct access link bypasses recipient filters, but it does not bypass parent portal login or the form's selected user type rules.

</Callout>

## What this link does

The direct access link helps you give selected families access to a form in special cases.

- Gives selected families a direct path to the form
- Still requires login on the parent portal
- Still follows the form's selected user types
- Expires automatically after the timeframe you pick

## Where to find it

Find the link option from inside the form's **Recipients** area.

<Steps>
  <Step title="Open the form" icon="folder" title-type="p">

  - Open the form in the admin portal.

  </Step>
  <Step title="Go to Recipients" title-type="p">

  - Select the **Recipients** section.

  </Step>
  <Step title="Find the Direct access link card" title-type="p">

  - Locate the **Direct access link** card in the **Recipients** area.

  </Step>
</Steps>

## How to generate a link

Create the link from the **Direct access link** card, then send it to the family who needs access.

<Steps>
  <Step title="Choose how long the link stays active" title-type="p">

  - Under **Valid for**, pick one duration: **7 days**, **14 days**, **30 days**, **60 days**, or **90 days**.

  </Step>
  <Step title="Generate the link" title-type="p">

  - Select **Generate link**.

  </Step>
  <Step title="Copy the link" title-type="p">

  - Select **Copy link**.

  </Step>
  <Step title="Send it to the family" title-type="p">

  - Share the copied link with the family who should use the form.

  </Step>
</Steps>

After you create the link, the card shows the expiration date and time so you can confirm how long it will remain available.

## Before you send the link

Check these items before you share the link:

- form is active or live
- form start and end dates are currently valid
- correct user type or user types are selected in **Recipients**

Wrong form settings can still prevent access, even when the link was created successfully.

## What families experience

From the family's side, the process is straightforward:

- open the link
- sign in on the parent portal if not already signed in
- see the form after sign-in if their user type is allowed

## Common use cases

Use a direct access link when you need an exception without changing your main recipient rules.

- late registrant needs immediate access to one form
- one-off exception where filters are too narrow
- support team needs a quick access path without changing the main filter rules

## Troubleshooting

<ExpandableGroup>
  <Expandable title="Could not generate link" default-open="false">

  Check that the form is active or live and that the form dates are valid right now. If the settings look correct, save the form and try again.

  </Expandable>
  <Expandable title="Family says link does not work" default-open="false">

  Confirm the family is using the correct parent portal, the link has not expired, and the correct user type is included for the form.

  </Expandable>
  <Expandable title="Family can log in but still cannot access form" default-open="false">

  The direct access link bypasses recipient filters only. User type restrictions still apply, so confirm that the family's user type is allowed on the form.

  </Expandable>
</ExpandableGroup>

## Best practices

Use direct access links as a backup option, not as your main recipient process.

- use the shortest duration that fits the need
- do not post direct links on public pages
- regenerate a new link if the old one expires
- keep recipient filters as the default process and use direct links for exceptions
