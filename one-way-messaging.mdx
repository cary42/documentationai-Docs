## What one-way messaging is

One-Way Messaging lets people in the same family account send short notes to another person in that same family, such as a parent sending a message to a child or one guardian sending a message to another guardian.

<Callout kind="info" collapsed="false">
  One-Way Messaging is not a reply thread or chat. Each message is sent as a single note, and Campium does not provide back-and-forth conversation within this feature.
</Callout>

Campium saves these messages so staff can review them later, search or filter the list, and print messages when needed. Families use the feature from the family site, and staff manage it from the admin site.

## Turn on one-way messaging

Enable the feature before families or staff can use it.

<Steps>
  <Step title="Enable the module" icon="settings" title-type="p">
    - Turn on the **One-Way Messaging Module** in your Campium by going to **Admin** -> **Site Settings** and enabling the **One-Way Messaging&#x20;**&#x6D;odule

    - If the module is off, families do not see the link on their account page and staff do not see the admin menu item.
  </Step>

  <Step title="Give staff permission" icon="users" title-type="p">
    - Assign the staff permission tied to **One-Way Messaging** to anyone who should review, filter, print, or manage messages. You can manage roles & permissions by going to Admin -> Roles & Permissions.

    - Staff without that permission cannot access the admin page.
  </Step>

  <Step title="Set the daily limit if needed" icon="bar-chart" title-type="p">
    - Use **One-Way Messages Per Day** to control how many messages a logged-in family user can send in one day.

    - Set the value to `0` if you do not want to enforce a daily limit.
  </Step>
</Steps>

Once the module is on, families can access One-Way Messaging from the family site and staff can open the admin page.

## Family site

Families use One-Way Messaging from the family subdomain at `/account/one-way-messaging/`. The logged-in person can send messages only to other people in the same family account.

<Tabs>
  <Tab title="Family site" icon="users">
    Families send and review their own one-way messages from `/account/one-way-messaging/` on the family site.

    The page shows only eligible recipients from the same family account. The logged-in person never appears as their own recipient.

    If the account has only one person, or no other eligible family members, the page shows that there is no one to send to.
  </Tab>

  <Tab title="Admin" icon="settings">
    Staff review and manage all one-way messages from `/one-way-messaging/` on the admin site.

    The admin page shows camp-wide message records, including sender, recipient, message text, send time, and printed or unprinted status.

    Staff can filter the list, print messages, change printed status, or delete a single message.
  </Tab>
</Tabs>

### Send a message on the family site

Use the family page to send a single note to another person in the same family.

<Steps>
  <Step title="Open One-Way Messaging" icon="folder" title-type="p">
    - Sign in to the family site.

    - Go to `/account/one-way-messaging/` on your family subdomain.

    - If the feature is enabled, the One-Way Messaging page opens.
  </Step>

  <Step title="Choose a recipient" icon="users" title-type="p">
    - Select one eligible person from the same family account.

    - The logged-in person does not appear in the recipient list.

    - If no eligible family members are available, the page shows that there is no one to send to.
  </Step>

  <Step title="Enter the message" icon="edit" title-type="p">
    - Type the short note you want to send.

    - Review the recipient before sending, since the feature does not create a reply thread after the message is sent.
  </Step>

  <Step title="Send the message" icon="play-circle" title-type="p">
    - Submit the message from the page.

    - After the message sends, it is saved in Campium and becomes available for staff review in admin.
  </Step>
</Steps>

### View message history on the family site

The family page also shows message history for the logged-in person. This history includes messages that person has sent, with the **date**, **from**, **to**, and **text** for each message.

<Callout kind="info" collapsed="false">
  The family history is not an inbox of received messages. It shows only messages sent by the logged-in person.
</Callout>

### Daily message limits

If your camp sets **One-Way Messages Per Day**, each logged-in family user can send only that many messages per day. When the person reaches the limit, they cannot send another message until the next day.

<Callout kind="alert" collapsed="false">
  If **One-Way Messages Per Day** is set to a value other than `0`, families may see that they cannot send more messages after reaching the daily limit. Set the value to `0` if you do not want a limit.
</Callout>

## Admin

Staff manage One-Way Messaging from `/one-way-messaging/` on the admin site. The page shows all one-way messages for your camp instance, not only messages from one family.

### What staff see

The main table includes these message details:

- **Sender**

- **Recipient**

- **Text**

- **Send time**

- **Printed or unprinted status**

This page is the staff record of one-way messages in Campium. Use it to review activity, narrow the list to the messages you need, and prepare printouts.

### Find messages and print them

Use filters and print options together when you need a specific group of messages.

<Steps>
  <Step title="Open the admin page" icon="folder" title-type="p">
    - Sign in to the admin site.

    - Go to `/one-way-messaging/`.

    - The page opens with the current message list for your camp instance.
  </Step>

  <Step title="Set filters" icon="settings" title-type="p">
    - Choose a **date range** to narrow the message list.

    - If needed, add the optional **profile or custom-field multiple-choice filter** based on the recipient.

    - You can choose to include all answers for a field and group the results by that segment.
  </Step>

  <Step title="Refresh or reset the list" icon="arrow-right" title-type="p">
    - Click **Apply Filter** to refresh the table with the current filter choices.

    - Click **Clear Filter** to return to the unfiltered view.
  </Step>

  <Step title="Choose print layout" icon="bar-chart" title-type="p">
    - Select whether to print **one message per page** or **all messages together within each group**.

    - If you grouped by a field segment, the printout follows that grouping.
  </Step>

  <Step title="Print unprinted messages" icon="play-circle" title-type="p">
    - Use **Print Unprinted Messages** to print unprinted messages that match the current filters.

    - After printing, Campium treats those matching messages as printed.
  </Step>
</Steps>

<Callout kind="alert" collapsed="false">
  `Print Unprinted Messages` changes message status as part of the print action. If staff need to print the same messages again as unprinted, they must unmark them first.
</Callout>

### Save your defaults

Use **Save my defaults** to remember the field and value choice and the print layout you want to reuse next time.

By default, the date range generally starts with **yesterday through today** unless a custom URL filter applies. Saving defaults helps staff return to the same filtering and printing setup without rebuilding it each time.

### Reprint or change printed status

Printed status is reversible for selected messages. If a message was already printed and you need to print it again as unprinted, select it and use **Unmark Selected as Printed**.

This changes those selected messages back to unprinted so they can be included in a later unprinted print run.

<Callout kind="alert" collapsed="false">
  Changing printed status affects whether messages appear in later unprinted print jobs. Confirm your selection before using **Unmark Selected as Printed**.
</Callout>

### Delete a message

Use **Delete** to remove a single message from the list.

<Callout kind="alert" collapsed="false">
  Delete removes one message from the admin list. Confirm that you are removing the correct record before deleting it.
</Callout>

## Family site and Admin at a glance

Use the family site to send messages. Use admin to review and manage the saved record of those messages.

| Area        | Path                          | What it shows                                                  | What people can do                                                                      |
| ----------- | ----------------------------- | -------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| Family site | `/account/one-way-messaging/` | The logged-in person's sending screen and sent-message history | Send a message to another person in the same family and view sent history               |
| Admin       | `/one-way-messaging/`         | All one-way messages for the camp instance                     | Review, filter, print, save defaults, change printed status, or delete a single message |

## Best practices and policy notes

Keep One-Way Messaging focused on short family-to-family notes inside the same account. Because the feature is one-way, it works best for simple communication that does not require a reply in Campium.

Set a daily limit only if your camp needs to control message volume. If you do use a limit, make sure families understand that the limit resets the next day.

Review printed status carefully when staff print messages in batches. Printed and unprinted states affect later print workflows, especially when staff rely on **Print Unprinted Messages** for daily processing.
